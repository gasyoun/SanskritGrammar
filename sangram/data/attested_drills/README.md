## `attested_drill_items.tsv` schema

| Column | Meaning |
|---|---|
| `lemma_id` | DCS lemma id (homonyms are separate rows, EM7) |
| `lemma` | IAST citation form |
| `stem_class` | Zaliznyak stem class ŌĆö `a-stem`, `i-stem`, `consonant-stem`ŌĆ” |
| `gender` | dominant `feat_gender` by token count |
| `cell` | the attested cell, `Case.Number` (e.g. `Ins.Plur`) |
| `corpus_count` | tokens of this lemma in this cell |
| `freq_band` | kosha `count_all` band ŌĆö `1000+` / `100-999` / `10-99` / `2-9` / `hapax` |
| `core_rank` | kosha core-curriculum rank (the ordering key) |
| `expected_form` | vidyut-prakriy─ü generation, SLP1, `\|`-separated |
| `attested_forms` | top unsandhied corpus forms, IAST, `\|`-separated |
| `agreement` | `match` / `variant` / `mismatch` / `no_attested_form` / `no_generation` |

## The rule this asset exists to enforce

**Generated-vs-attested disagreements are flagged, never silently picked.** Where the
paradigm engine and the corpus differ, the row carries both sides and names the
relationship. `variant` rows are real Vedic/classical doublets (`deva` Ins.Plur
attests both *devaißĖź* and *devebhißĖź*; vidyut generates only the former). `mismatch`
rows are excluded from drilling entirely ŌĆö a cell whose generated form is unattested
has no authoritative answer to grade a learner against.

`mismatch` also turns out to be a **DCS annotation-error detector**: `artha` Voc.Sing
attests *arthaißĖź* (n=6), which is an instrumental plural, not a vocative. That is a
corpus mis-annotation surfaced by the flag, not a fact about Sanskrit.

## Regenerate

```sh
python sangram/data/attested_drills/build_attested_declension_drills.py
```

~11 minutes, dominated by the DCS full scan. To retune only the presentation layer
(widget quotas, report shape) without re-deriving corpus evidence:

```sh
python sangram/data/attested_drills/build_attested_declension_drills.py --widget-only
```

~13 seconds; reproduces the report byte-identically from the committed TSV.

## Limits

- Comparison uses `m_unsandhied`, never the raw surface `form` (sandhi would register
  as paradigm disagreement).
- Stem class is a citation-form approximation, not a paradigmatic classification ŌĆö
  the [SG-MO-001 ┬¦ 6.1](https://github.com/gasyoun/SanskritGrammar/blob/main/sangram/articles/declension-overview/index.mdx)
  caveat. 52 lemmas the index calls `indeclinable` nevertheless carry case-marked DCS
  tokens; that is a cross-source disagreement, not a language fact. **The TSV keeps
  them; the trainer excludes the whole class** ŌĆö the contradiction is evidence worth
  recording, but presenting it as an exercise would teach that indeclinables decline.
- Coverage here (mean 7.72 cells of 24) is far above G2's headline 10.44 % because
  this asset is restricted to kosha *core* lemmas, which are attested more fully by
  definition. Both numbers are correct; the denominators differ.
- A lemma enters the corpus only if it is present in **all three** join inputs
  (30 878 of 57 144 coverage lemmas; 5 838 after the `core_rank` restriction).
