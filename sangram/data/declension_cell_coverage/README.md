## Headline (evidence-limit EM3 at corpus scale)

Over **57 144** noun lemmas / **1 790 270** case×number tokens (denominator matches
[SG-MO-001](https://github.com/gasyoun/SanskritGrammar/blob/main/sangram/articles/declension-overview/index.mdx)):

- **median 1** attested cell per lemma; **58.9 %** of lemmas attest exactly one cell; **0.0 %** attest all 24.
- only **10.44 %** of the possible lemma×cell space is ever attested.
- coverage is a **frequency artifact, not a lemma property**: mean cells/lemma rises 1.0 (hapax) → 1.93 (2–9) → 5.55 (10–99) → 11.62 (100–999) → **17.32 (1000+)**, and even at 1000+ only 0.63 % attest the full paradigm.
- cell asymmetry: **Nom.Sing** attested by 41 998 lemmas vs **Dat.Dual** by 128 — a ~328× spread.

This confirms and generalises pilot P1's a-stem finding (median 1) across the whole noun inventory: the "full paradigm" is a generated/traditional object, almost never an attested one — every declension article must state denominators, never imply per-lemma completeness.

## Limits

1. **EM3** — attested cells per lemma is almost always ≪ 24; this asset quantifies it, it does not fill it.
2. **`stem_final` is an approximation** — lemma-ending only (citation form ≠ stem, [SG-MO-001 §6.1](https://github.com/gasyoun/SanskritGrammar/blob/main/sangram/articles/declension-overview/index.mdx)); a convenience filter tag, **not** a paradigmatic classification. Each article does its own precise stem-typing (e.g. SG-MO-001's attested-Ins.Sg method).
3. **EM7** — homonyms keyed by `lemma_id`; one citation string with two ids is two rows.
4. **Pin** — orphaned `04e0778`, bound as above.
