# Zaliznyak-made-learnable — the on-ramp (design)

_Created: 14-07-2026 · Last updated: 14-07-2026_

Design of a **gentler, graded entry** into Zaliznyak's Ряд/Тип/seṭ verbal-root classification —
the "map before the territory" §0 that [`IMPROVEMENT_PLAN.md`](https://github.com/gasyoun/SanskritGrammar/blob/main/TolchelnikovTalmud_2026/IMPROVEMENT_PLAN.md)
Phase 1 called for — sitting **before** Tolchelnikov's *Талмуд* in the ladder. The Талмуд stays the
deep-dive tier; the on-ramp gets a Kochergina-stage learner to where they can read **one** derivation,
then hands off "one tap deeper." Wave-1c of the
[digital-Sanskrit-pedagogy field](https://github.com/gasyoun/SanskritGrammar/blob/main/DIGITAL_SANSKRIT_PEDAGOGY_FIELD_2026.md)
(aspect §3.6), handoff [H915](https://github.com/gasyoun/Uprava/blob/main/handoffs/H915-Opus_SanskritGrammar_pedagogy-w1c-zaliznyak-onramp_14.07.26.md).

**Why.** The Талмуд itself states that *notation is a barrier taller than the Sanskrit it describes*.
The on-ramp removes that barrier for first contact: minimal notation, four rows, real high-frequency
roots, the same widgets the learner will meet again in the full text.

## The gentle path (what it teaches)

- **The big idea** — roots come in a few *behavioural families*; the family predicts the forms (you compute, you don't memorise a table per root).
- **Step 1 — three grades** (слабая / гуна / вриддхи), on the **4 high-frequency rows A₁ / I₁ / U₁ / R₁ only** (AblautMachine, rows-filtered).
- **Step 2 — position + type** — the ending imposes a *position*; the root's *type* (s / a / v) says how obediently it moves. Base types only, stated as three plain rules.
- **Step 3 — the connecting -i-** (seṭ / aniṭ binary) on DCS-top roots (SetTree), then the bridge into the full Талмуд.

## Widget reuse

`AblautMachine` gained two optional, **defaulted** props (`rows` whitelist, `initialSeries`) so the
on-ramp shows only A₁/I₁/U₁/R₁ while `<AblautMachine />` in talmud-02 still renders all 13 rows,
byte-identical. `SetTree` is reused as-is (already a minimal seṭ/aniṭ binary tree). No new widget code.
