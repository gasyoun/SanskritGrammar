# Zaliznyak-made-learnable — on-ramp design (W1c)

_Created: 14-07-2026 · Last updated: 14-07-2026_

The **on-ramp** is a short, graded, minimal-notation path into the Ряд → Тип → seṭ apparatus that
the [Толчельников *Талмуд*](https://github.com/gasyoun/SanskritGrammar/tree/main/TolchelnikovTalmud_2026)
formalises in full. The Талмуд is often *overkill* for a first pass — its generative notation
(`{√akṣ} + {2.syø}`), the 13-row A₁…N₂ series matrix, and the veṭ v1–v4 subclasses are exactly the
"утону в индексах и таблицах" ("drowning in indices and tables") load
[`student-roadmap.md`](https://github.com/gasyoun/SanskritGrammar/blob/main/TolchelnikovTalmud_2026/student-roadmap.md)
warns about. The on-ramp is the gentler front door; the Талмуд stays as the deep-dive tier behind it
(**progressive disclosure** — every on-ramp step carries a "one tap deeper" link into its full chapter).

## The core idea — three questions, not seven steps

The Талмуд's Master Pipeline (Раздел VIII) is a 7-step generative algorithm. The on-ramp keeps only
the **three questions** that a learner must internalise before the pipeline makes any sense, in order:

1. **Какая ступень?** (Which grade?) — Слабая / Гуна / Вриддхи. The ending asks for one of the three.
2. **Слушается ли корень?** (Does the root obey?) — its **Тип** (`s` / `a` / `v`) decides whether it
   actually reaches that grade or resists.
3. **Нужна ли `i`?** (Is a connecting `-i-` inserted?) — the **seṭ / aniṭ** binary.

Reduplication, samprasāraṇa, inter-Ряд movement, and sandhi (Талмуд chapters IV–VII) are **not** part
of the on-ramp — they are the pipeline's later steps and stay one tap deeper.

## Simplification rules (what is deferred, what stays)

The on-ramp is defined by what it *omits*. Every omission is a deliberate deferral, not a gap.

| Concept | On-ramp (kept, minimal) | Deferred to the Талмуд (one tap deeper) |
|---|---|---|
| **Ступень** (grade) | The three grades named plainly: Слабая · Гуна · Вриддхи | The full позиция 1/2/3 → grade formalism |
| **Ряд** (series) | Only the **4 highest-frequency rows** — A₁, I₁, U₁, R₁ — as a pocket table + the live `AblautMachine` | The full 13-row A₁…N₂ matrix (Таблица 2), before-vowel allomorphs, syllabic sonants (`m̥`, `n̥`) — chapter II |
| **Тип** (type) | The 3 plain behaviours: `s` obeys · `a` resists Вриддхи · `v` resists Гуна | The v1–v4 subclasses and their specific modifications (Таблица 3) — chapter II |
| **seṭ** | The binary yes/no + the 3-question decision tree, via the live `SetTree` | veṭ (`v1–v4`) and the full junction record `{A} + {B}` (§§ 64–68) — chapter III |
| **Notation** | Plain Latin transliteration + inline backticks (`kṛ`, `guṇa`, `seṭ`) | The MST generative record `{√Root} + {Suffix}` (Раздел I / VIII) |
| **Pipeline** | The 3 questions as a linear "собери форму" recap | The full 7-step Master Pipeline (Раздел VIII) |

**Rule of thumb encoded here:** the on-ramp shows *one live widget per mechanism* and *one worked
example per rule*; anything that needs a second table or an exception list is deferred by a link.

## Page structure

Graded sequence of four `.mdx` pages under
[`TolchelnikovTalmud_2026/onramp/`](https://github.com/gasyoun/SanskritGrammar/tree/main/TolchelnikovTalmud_2026/onramp),
grouped in their own sidebar sub-category (`_category_.json`, `position: 0`) so they sit **before**
Талмуд chapter 00 in the ladder. Docs are a single autogenerated Docusaurus instance
(`routeBasePath: grammars`), so the pages appear automatically; no plugin wiring.

| Page | Question | Reuses | One tap deeper |
|---|---|---|---|
| `onramp-00-start.mdx` | The 3-questions map + how to read the notation | — | overview links to all three chapters |
| `onramp-01-ryad.mdx` | Ступень + Ряд (Q1) | `AblautMachine` widget; 4-row pocket table | `../talmud-02-cheredovanie` · Zaliznyak §§ 50–57, 60, 62 |
| `onramp-02-set.mdx` | seṭ / aniṭ (Q3) | `SetTree` widget; 3-question decision tree | `../talmud-03-soedinitelnaya-i` · Zaliznyak § 63 |
| `onramp-03-tip-next.mdx` | Тип (Q2) + собери форму + куда дальше | Таблица 3 (3 rows) | `../talmud-08-algoritm`, `../talmud-appendix-1`, `../zalizniak-concordance` · Zaliznyak §§ 47–49 |

Q2 (Тип) is deliberately placed **last**: it only becomes meaningful once the learner has seen a grade
(Q1) and knows a root can carry hidden behaviour, so the on-ramp teaches "grade → obedience" rather
than the Талмуд's позиция-first order.

## Progressive-disclosure link map

Each on-ramp step is a leaf that fans *down* into the Талмуд, never a dead end:

- **Within the on-ramp:** `onramp-00 → onramp-01 → onramp-02 → onramp-03` (relative `./onramp-0N-…` links).
- **Into the Талмуд:** relative extension-less links (`../talmud-02-cheredovanie`), resolved at build
  time — `onBrokenLinks: 'throw'` guarantees no on-ramp link rots silently.
- **Into Zaliznyak:** the existing
  [`ZRef`](https://github.com/gasyoun/SanskritGrammar/blob/main/src/components/ZRef.jsx) chip
  (`<ZRef sec="…"/>`), reusing the same §-anchored *Очерк* targets the Талмуд chapters already use.
