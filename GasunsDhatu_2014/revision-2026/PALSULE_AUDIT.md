# Аудит C10/L1 — Пальсуле-центризм конкорданса против цифровой дхатупатхи

_Created: 07-07-2026 · Last updated: 08-07-2026_

**Претензия (С. А. Крылов, отзыв ведущей организации, 2014; корробурирует дефект L1
ревизии H246):** конкорданс диссертации (Приложение 3) выбрасывает корни, которых нет у
Пальсуле, — например `4añc`, хотя он есть у Панини, — но при этом держит корни,
засвидетельствованные только у Пальсуле (`2and`, `ast`).

**Проверка (07-07-2026, Fable 5 `claude-fable-5`)** против цифровой дхатупатхи vidyut —
[WhitneyRoots/scratch/vidyut_data/prakriya/dhatupatha.tsv](https://github.com/gasyoun/WhitneyRoots/blob/main/scratch/vidyut_data/prakriya/dhatupatha.tsv)
(2 259 дхату, кодировка gaṇa.sūtra, надмножество пяти традиционных дхатупатх; SLP1).

| Корень | Что нашлось в vidyut | Вердикт по претензии |
|---|---|---|
| `añc` | **Пять** дхату семейства añc/ac: `01.0215 ancu~ gatipUjanayoH` (bhvādi); `01.0998 ancu~^`, `01.0999 acu~^`, `01.1000 aci~^` — все `gatO yAcane ca`; `10.0266 ancu~ viSezaRe` (curādi) | **Подтверждено.** Паниниевская традиция несет несколько омонимичных añc; выброшенный из-за отсутствия у Пальсуле `4añc` восстановим из цифровой дхатупатхи. В самом конкордансе (§3.4, строка «añc») учтено меньше омонимов. |
| `and` | Ни одного дхату на `and-` | **Согласуется с Крыловым:** `2and` — действительно только-Пальсуле относительно паниниевской дхатупатхи. |
| `ast` | `10.0169 asta~ saNGAte` (curādi) есть | **Уточнение к отзыву:** второй пример Крылова неточен — `ast` в паниниевской (цифровой) дхатупатхе ЕСТЬ, т.е. «только-Пальсуле» он не является. |

**Итог.** Риск L1/C10 реален и теперь механически проверяем: цифровая дхатупатха 2026 г.
позволяет пройти по всем исключениям конкорданса и вернуть «потерянные настоящие корни».

**Метод полной сверки (задел, в этой сессии НЕ выполнялся — конкорданс не дописывается
без автора):**

1. Оцифровать колонку исключений Приложения 3 (в `.mdx` вошли только «Материалы для
   конкорданса» §3.4; полный конкорданс — в исходном `.docx`/PDF диссертации).
2. Нормализовать записи в SLP1 (транскодеры `sanskrit-util`, см.
   [SHARED_CODE.md](https://github.com/gasyoun/github-spine/blob/main/SHARED_CODE.md)).
3. Джойн против `dhatupatha.tsv` (vidyut) и
   [WhitneyRoots/crosswalk/roots.csv](https://github.com/gasyoun/WhitneyRoots/blob/main/crosswalk/roots.csv):
   всё, что есть у Панини (vidyut) и/или у Уитни, но отсутствует в конкордансе из-за
   Пальсуле, — кандидат в «потерянные настоящие корни».
4. Список кандидатов — на визу автору; естественный дом для этой сверки — статья A39
   («Grammar, dictionary, corpus — where the three disagree», Uprava/ARTICLES.md).

## Задел H328 (08-07-2026, Fable 5 `claude-fable-5`) — измерение шага 3 метода

Пробный джойн шага 3 выполнен на готовой паре датасетов, БЕЗ оцифровки конкорданса:
[WhitneyRoots/crosswalk/roots.csv](https://github.com/gasyoun/WhitneyRoots/blob/main/crosswalk/roots.csv)
(930 корней) × [WhitneyRoots/src/paradigms.json](https://github.com/gasyoun/WhitneyRoots/blob/main/src/paradigms.json)
(it-stripped сопоставление с дхатупатхой vidyut, метод «clean root на последнем it-lopa»,
готовый артефакт — переиспользован, не пересчитывался).

- Совпало по чистой it-stripped форме: **454 из 930** корней Уитни; не совпало 476.
- **Отрицательный результат, важный для метода:** список «несовпавших» НЕ является списком
  кандидатов в «потерянные настоящие корни» — его возглавляют бесспорно паниниевские
  корни (√i «идти», √ah «говорить», √an «дышать»), не совпавшие из-за аблаута и различий
  формы цитирования (i/ay/e; акшары с назализацией), а не из-за отсутствия в дхатупатхе.
- Следствие: шаг 3 требует нормализации ступеней чередования (или джойна через
  леммы DCS / номера MW), прежде чем давать кандидатов на визу. Наивный джойн по
  равенству дает преимущественно ложные срабатывания — не тратить на него визу автора.

Список кандидатов для А39 по-прежнему заблокирован шагом 1 (оцифровка колонки
исключений Приложения 3 из исходного `.docx`/PDF).
