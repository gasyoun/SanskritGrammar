## 🧠 Dev Notes & Hypotheses (Bugs, ideas, context)
- **Critical MDX gotcha, found the hard way (06-07-2026, Sonnet 5 `claude-sonnet-5`):** Pandoc
  backslash-escapes any literal `<`/`>` in source prose (comparative-linguistics notation like
  `a < ā`, or garbage like a literal `<TBODY>` pasted into the original Word doc) as `\<`/`\>`.
  MDX's JSX tokenizer does **not** reliably respect that markdown escape and can crash the
  **entire page's** compile with "Unexpected closing slash, expected an open tag first" — and
  because the `rstTable` plugin re-parses each table cell's text through the same MDX-aware
  parser, this bites inside tables too, not just prose. One stray `\<` (a Greek etymology
  "σκέπτομαι \< \*σπεκ‑" in `ZalizniakKonspekt_2004`) broke that file's entire build. A scan of
  all 6 sources found **200 total instances across 4 of the 6 files** — silently present since an
  earlier conversion pass that was never build-verified end-to-end. Fixed by converting every
  `\<`/`\>` to `&lt;`/`&gt;` (inert HTML entities, safe in MDX, losslessly preserves the real
  character) — now baked into `/docx-to-md` as a mandatory post-process step
  (`escape_stray_angle_brackets`).
- `BuhlerLeitfaden_1923/Buhler_Unicode.mdx` (like all 6) uses Pandoc's grid-table syntax for
  merged/spanned cells, wrapped in a plain ```` ```rst-table ```` fenced block; rendered by
  `src/remark/rstTable.mjs` (ported verbatim from
  [buhler-sanskrit-book](https://github.com/gasyoun/buhler-sanskrit-book/blob/main/src/remark/rstTable.ts),
  the canonical original — also ported into `csl-guides/src/remark/rstTable.mjs`). Pure JS, no
  Pandoc dependency at Docusaurus build time, true `rowSpan`/`colSpan` (confirmed: 21 `colspan`
  attributes render correctly in the built `BuhlerLeitfaden_1923` page).
- Docusaurus config (`docusaurus.config.mjs`) serves the 6 `.mdx` files **directly from their
  own per-book folders** (`docs.path: '.'`, an `include` glob per folder) — no copy into a
  `docs/` directory. `onBrokenLinks`/`onBrokenMarkdownLinks` are `'warn'`, not `'throw'`: the
  auto-generated Cyrillic heading anchors produce some broken internal TOC links (pre-existing
  content issue, not introduced by this session — not yet triaged).
