  pushed). A **shared-index race** on a *bare* `git commit` also meant only 3 of 7 mdx were
  captured. Recovery: regenerated all 7 mdx from the untouched sources, then delivered via
  **explicit-pathspec commits in an isolated `git worktree`** (this .ai_state edit included) вЂ” the
  only reliable way to work while the other session keeps stealing the shared HEAD. Lessons,
  reinforced: (1) with a concurrent session, **isolate in a worktree** вЂ” never edit/commit the
  shared tree; (2) **always commit with an explicit pathspec**, never bare (`git commit -- <paths>`),
  or a racing index truncates the commit; (3) EmEditor is an active watcher here too вЂ” it reverted
  `docusaurus.config.mjs`/`.ai_state.md` edits repeatedly, so land+commit atomically.
- **5 more MDX-crash Pandoc-isms found converting books 7 & 8 (06-07-2026, Opus 4.8 `claude-opus-4-8`),
  all now fixed in `/docx-to-md`'s `make_mdx_safe`:** (1) `<!-- -->` list-separator comments вЂ” MDX
  rejects HTML comments; (2) `<user@host>` / `<url>` autolinks вЂ” `<` opens JSX, `@`/`:` illegal in a
  tag name в†’ rewrite to `[x](mailto:x)`/`[x](x)`; (3) `<u>` styling on a hyperlink's text yields
  `<u>[text</u>](url)`, tags interleaving the link brackets в†’ strip `<u>` when its span holds a
  bracket; (4) **`{вЂ¦}` braces INSIDE rst-table cells** вЂ” the sanitizer only escaped braces OUTSIDE
  fences, but remarkRstTable re-parses each cell as MDX, so `{tГё}`-style notation (1360 cell lines in
  Talmud-2.1.6) fails acorn; (5) **span-tag inflation overflowing a fixed-width grid column** вЂ”
  `[x]{.mark}` в†’ `<mark>x</mark>` adds 13 chars, and a 17-char `<mark>turv</mark>` in a 15-wide Pandoc
  column gets sliced mid-tag by the column-position-based rst parser в†’ "Unexpected end of file in
  name". Fix: inside table cells STRIP `.mark`/`.underline` to bare text. Lesson reinforced: **only
  `npm run build` proves MDX validity** вЂ” each passed the grep pre-checks, all failed at compile.
- **Critical MDX gotcha, found the hard way (06-07-2026, Sonnet 5 `claude-sonnet-5`):** Pandoc
  backslash-escapes any literal `<`/`>` in source prose (comparative-linguistics notation like
  `a < ДЃ`, or garbage like a literal `<TBODY>` pasted into the original Word doc) as `\<`/`\>`.
  MDX's JSX tokenizer does **not** reliably respect that markdown escape and can crash the
  **entire page's** compile with "Unexpected closing slash, expected an open tag first" вЂ” and
  because the `rstTable` plugin re-parses each table cell's text through the same MDX-aware
  parser, this bites inside tables too, not just prose. One stray `\<` (a Greek etymology
  "ПѓОєО­ПЂП„ОїОјО±О№ \< \*ПѓПЂОµОєвЂ‘" in `ZalizniakKonspekt_2004`) broke that file's entire build. A scan of
  all 6 sources found **200 total instances across 4 of the 6 files** вЂ” silently present since an
  earlier conversion pass that was never build-verified end-to-end. Fixed by converting every
  `\<`/`\>` to `&lt;`/`&gt;` (inert HTML entities, safe in MDX, losslessly preserves the real
  character) вЂ” now baked into `/docx-to-md` as a mandatory post-process step
  (`escape_stray_angle_brackets`).
- `BuhlerLeitfaden_1923/Buhler_Unicode.mdx` (like all 6) uses Pandoc's grid-table syntax for
  merged/spanned cells, wrapped in a plain ```` ```rst-table ```` fenced block; rendered by

- Docusaurus config (`docusaurus.config.mjs`) serves the 6 `.mdx` files **directly from their
