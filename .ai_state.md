  `docusaurus.config.mjs`/`.ai_state.md` edits repeatedly, so land+commit atomically.
- **5 more MDX-crash Pandoc-isms found converting books 7 & 8 (06-07-2026, Opus 4.8 `claude-opus-4-8`),
  all now fixed in `/docx-to-md`'s `make_mdx_safe`:** (1) `<!-- -->` list-separator comments — MDX
  rejects HTML comments; (2) `<user@host>` / `<url>` autolinks — `<` opens JSX, `@`/`:` illegal in a
  tag name → rewrite to `[x](mailto:x)`/`[x](x)`; (3) `<u>` styling on a hyperlink's text yields
  `<u>[text</u>](url)`, tags interleaving the link brackets → strip `<u>` when its span holds a
  bracket; (4) **`{…}` braces INSIDE rst-table cells** — the sanitizer only escaped braces OUTSIDE
  fences, but remarkRstTable re-parses each cell as MDX, so `{tø}`-style notation (1360 cell lines in
  Talmud-2.1.6) fails acorn; (5) **span-tag inflation overflowing a fixed-width grid column** —
  `[x]{.mark}` → `<mark>x</mark>` adds 13 chars, and a 17-char `<mark>turv</mark>` in a 15-wide Pandoc
  column gets sliced mid-tag by the column-position-based rst parser → "Unexpected end of file in
  name". Fix: inside table cells STRIP `.mark`/`.underline` to bare text. Lesson reinforced: **only
  `npm run build` proves MDX validity** — each passed the grep pre-checks, all failed at compile.
- **Critical MDX gotcha, found the hard way (06-07-2026, Sonnet 5 `claude-sonnet-5`):** Pandoc
  backslash-escapes any literal `<`/`>` in source prose (comparative-linguistics notation like
  `a < ā`, or garbage like a literal `<TBODY>` pasted into the original Word doc) as `\<`/`\>`.
  MDX's JSX tokenizer does **not** reliably respect that markdown escape and can crash the
  **entire page's** compile with "Unexpected closing slash, expected an open tag first" — and
  because the `rstTable` plugin re-parses each table cell's text through the same MDX-aware
  parser, this bites inside tables too, not just prose. One stray `\<` (a Greek etymology
  "σκέπτομαι \< \*σπεκ‑" in `ZalizniakKonspekt_2004`) broke that file's entire build. A scan of
  all 6 sources found **200 total instances across 4 of the 6 files** — silently present since an
  earlier conversion pass that was never build-verified end-to-end. Fixed by converting every
  `\<`/`\>` to `&lt;`/`&gt;` (inert HTML entities, safe in MDX, losslessly preserves the real
  character) — now baked into `/docx-to-md` as a mandatory post-process step
