## Аудит морфологических разборов (H797, фаза 2 — развилка метода)

Третья книга конвейера проверки утверждений — и первая, где конвейер пришлось
**адаптировать под жанр**. [H797](https://github.com/gasyoun/Uprava/blob/main/handoffs/H797-Fable_SanskritGrammar_claim-verification-backlog-verify-and-cross-grammar-generalise_12.07.26.md)
предписывает перенести на Кнауэра двухосевой конвейер Кочергиной/Бюлера
([KocherginaUchebnik_1998/README.md](https://github.com/gasyoun/SanskritGrammar/blob/main/KocherginaUchebnik_1998/README.md),
[BuhlerLeitfaden_1923/README.md](https://github.com/gasyoun/SanskritGrammar/blob/main/BuhlerLeitfaden_1923/README.md)),
который жнёт **дискурсивные** утверждения («всегда», «обычно», «большинство корней…») и
судит их по двум осям — факт и подача. Но оцифрованный текст Кнауэра — не дискурсивная
грамматика, а фразовая хрестоматия: жать в нём нечего, универсальных/частотных
формулировок в тексте нет (проверено по источнику до начала работы, 15-07-2026).
По запросу сессии автор постановил: **адаптировать** (а не пропустить книгу и не
ограничиться отчётом).

**Адаптированная единица анализа:** вместо риторического утверждения аудируется *разбор
одной сноски* — лингвистически ли корректен заявленный анализ поверхностной формы
(корень + категория + преверб) по Уитни 1889? Собственные §-номера Кнауэра указывают в
неоцифрованный том-компаньон и напрямую не сверяемы; сверяемо само лингвистическое
утверждение. Оси подачи здесь нет — это аудит **корректности разбора**, не
частотности/педагогики; DCS используется только как дополнительная проверка
засвидетельствованности, не как основная опора.

Файлы реестра:

- [parse_audit.yml](https://github.com/gasyoun/SanskritGrammar/blob/main/KnauerFrazy_1908/parse_audit.yml) — верифицированный реестр (KN-1..KN-10), источник правды; вердикты `CONFIRMED · QUESTIONABLE · ERROR`;
- [PARSE_AUDIT.md](https://github.com/gasyoun/SanskritGrammar/blob/main/KnauerFrazy_1908/PARSE_AUDIT.md) — рукописная сводка (генератора в духе `build_claims.py` пока нет — реестр мал);
- `block_backlog` внутри `parse_audit.yml` — **сырой текст сносок всех 19 наборов, сохранён дословно** (нулевая потеря информации): будущая работа извлекает из него отдельные разборы.

Намеренно **не** схема `claims.yml`: реестр не подхватывается `build_claims.py` и не
считается в планку «≥ 50 проверенных», определённую для дискурсивных утверждений, —
мера «сделано» для аудита разборов требует отдельного решения человека.

## Итог: 214 разборов, полный слив (15-07-2026)

**210 CONFIRMED · 4 QUESTIONABLE · 0 ошибок.** Посев (KN-1..KN-10, Sonnet 5) слит в полный
аудит пятью параллельными агентами (Fable 5, H797): каждый поддающийся аудиту разбор всех
19 блоков извлечён из дословного `block_backlog` (поверхностные формы восстановлены по
исходному mdx) и сверен с Уитни; ~82 сноски-голые-§-ссылки пропущены по правилу гайда
(морфологического утверждения не несут). `PARSE_AUDIT.md` теперь генерируется —
[build_parse_audit.py](https://github.com/gasyoun/SanskritGrammar/blob/main/KnauerFrazy_1908/build_parse_audit.py),
тот самый генератор, который заголовок посева предвещал.

### Главные выводы

1. **Аппарат сносок Кнауэра чист: ни одной доказуемой ошибки разбора на все 214.** Прогноз
   посева подтвердился на двадцатикратной выборке: глосса, выверенная автором по одному
   примеру перед глазами, имеет узкую поверхность ошибки — профиль надёжности фразовой
   хрестоматии 1908 года зеркален калибровочным проблемам дискурсивных грамматик.
2. **Четыре QUESTIONABLE — пробелы цитирования, не подозрения:** корни gaṇ и kṣal
   отсутствуют в оцифрованном Уитни и каталоге Талмуда; одна помета «part.» против личной
   формы kupyate — вероятный артефакт оцифровки; один каузативный stem не заверен в
   оцифрованном справочнике.
3. **Два трофея из follow-up посева:** kṛṣṇāyate закреплён за Уитни §1059b-c и оказался
   **гапаксом корпуса — единственное вхождение ровно в том стихе Хитопадеши, который Кнауэр
   и выписал** (он снабдил гапакс верным разбором продуктивной модели); вопрос о labh как
   «депоненте» растворён опорой D-B — каталог Талмуда даёт pada=U, сверхпрочтение посева
   снято, а собственное (верное) утверждение сноски подтверждено. Урок: аудируй сноску, а
   не свой пересказ сноски.
4. **Побочные находки данных:** строка Nr. 7 в `block_backlog` искажена относительно
   исходного mdx (агент аудировал по источнику; дефект помечен в баннере); в локусе
   посевного KN-3 исправлена ошибка самоссылки (Nr. 12 → Nr. 11).
5. **Извлечённый машиночитаемый инвентарь** — 214 разборов (форма + корень + категория +
   вердикт) — готовый датасет для сверки с сегментатором kosha и каталогом Талмуда.
